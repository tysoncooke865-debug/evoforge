# docs/exec — the executive layer

Product, priority and health documents. `HANDOVER.md` remains the engineering
door: *how* to work here. These answer *what to work on, and why*.

| File | What it is |
|---|---|
| `EXECUTIVE_REPORT_2026-07-24.md` | The founding report. Health score, funnel, the biggest risk and opportunity, top 5 priorities, department tasks — every figure queried live from production. **Read this first.** |
| `ARCHITECTURE_CURRENT.md` | The real 2026-07-24 architecture. Root `ARCHITECTURE.md` describes the retired Streamlit app and is history. |
| `ROADMAP_2026-07.md` | Now / Next / Later with ROI, effort and dependencies, plus a rejected-ideas list. Supersedes root `ROADMAP.md`. |
| `DASHBOARD_ARCHITECTURE.md` | Proposal for the executive dashboard. Nothing built; four open questions for Tyson at the end. |

## The one thing to know

27 real people have signed up. 10 ever logged a set. 2 have trained on four or
more days. The engineering is excellent and the funnel is broken — so the
roadmap puts activation, observability and return mechanics ahead of more depth.

## House rules these documents follow

- **Every number is queried, never estimated.** If it is an estimate it says so.
- **Say what was not done.** Each document ends with its own gaps.
- **A system without a backend is HIDDEN, never mocked** — including on dashboards.
- **Incidents are described by behaviour, never by identity.** This repo is public.
