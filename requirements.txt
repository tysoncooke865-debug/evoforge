# PIN EVERYTHING. And run Streamlit Cloud on Python 3.13.

# ############################################################################
# #  CLOUD MUST RUN PYTHON 3.13.                                             #
# #  Manage app -> Settings -> Advanced -> Python version -> 3.13.           #
# #  Every pin below then installs from a prebuilt wheel: no compiler, no    #
# #  source build. It is also what CI tests and what the app is developed    #
# #  against. This is not a preference; two outages came from ignoring it.   #
# ############################################################################
#
# Two outages on 2026-07-10, one root cause: Cloud had defaulted to Python 3.14,
# for which the native scientific stack barely has wheels.
#
# 1) UNPINNED. A redeploy re-resolved the whole stack and the app died before it
#    ran a line of its own code:
#
#        Uvicorn server started on :::8501
#        run-streamlit.sh: line 9: 177 Segmentation fault
#
#    A segfault at import time is a NATIVE ABI mismatch: pandas, numpy and pyarrow
#    all ship compiled C and must agree with each other. It is not a Python
#    exception, nothing can catch it, and it would have happened on ANY redeploy.
#    The commit that triggered it was innocent -- reverting it did not help.
#
# 2) PINNED, still on 3.14. numpy 2.2.6 publishes no cp314 wheel, so pip tried to
#    build it from source and Cloud's image has no compiler:
#
#        Downloading numpy-2.2.6.tar.gz (20.3 MB)
#        error: command 'cmake' failed: No such file or directory
#
#    First Linux cp314 wheels: numpy 2.4.4, pyarrow 22.0.0, pandas 3.0.x -- i.e.
#    the only versions installable on 3.14 are the bleeding-edge ones that
#    segfaulted in (1). Do not try to pin your way around a too-new interpreter.
#
# Fixed by setting Cloud's Python to 3.13. The app came back on the pins below.
#
# An unpinned requirements.txt means the app you test is not the app that deploys.
# Bump these deliberately, one at a time, and reboot Cloud to check.
# CI installs this file on Linux for 3.11 and 3.13 and executes app.py through
# AppTest -- the same import path that segfaulted. Green CI before every reboot.
